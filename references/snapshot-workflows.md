# GHL Snapshot Workflows — Copy Map

> Every client gets these workflows imported via the MetaTech/RizzDial snapshot.
> This document defines what copy each workflow needs.
>
> **IMPORTANT:** The skill generates copy for workflows ONLY if the client marked them GENERATE in the Existing Automations Inventory (SKILL.md input 5). Workflows marked KEEP are skipped entirely.
>
> **NEVER generate voicemail scripts for any workflow.** RizzDial agents run with `double_call: true` and voicemail detection disabled. The double call catches the missed-call moment; voicemails waste per-send budget.

---

## SECTION 0 (ALWAYS GENERATED): CALL OUTCOME TAG SCHEMA

This is the backbone of every playbook. Generated first, before any workflow copy.

Canonical tags (lowercase, hyphenated, no spaces):

| Tag | When Applied | What It Triggers |
|-----|--------------|------------------|
| `ai-agent-booked` | Lead books appointment | Removes from dialer → starts post-booking |
| `ai-agent-transferred` | Lead accepts live transfer | Removes from dialer → closer notification |
| `ai-agent-disqualified` | Fails qualification | Removes from dialer |
| `ai-agent-do-not-contact` | Explicit DNC / STOP | Removes from ALL sequences |
| `ai-agent-no-answer` | Both dials unanswered (analytics only) | Continues next day |
| `ai-agent-bad-number` | Wrong / disconnected number | Removes from dialer |
| `ai-agent-callback-requested` | Specific time requested | Pauses until time |
| `ai-agent-exhausted` | 14-day cap reached with no outcome | Removes from dialer |

Client's existing removal automations should listen for all except `no-answer` and `callback-requested`.

## WORKFLOW 01: AI DIAL SEQUENCE (ALWAYS GENERATED)

**GHL Name:** 01 -- AI Dial Sequence (double_call + pre-call SMS)
**Trigger:** Tag `new lead` (or whatever tag the client uses for leads needing Blake to call)
**Primary channel:** AI voice call via RizzDial webhook with `double_call: true`
**Support:** Pre-call SMS 60s before the daily webhook

### Architecture

```
DAY 1 (within business hours of opt-in)
  Minute 0:  Pre-call SMS sends
  Minute 1:  RizzDial webhook fires with double_call:true
             |
             Agent dials once
             |
             +- ANSWERED > agent qualifies > outcome tag > STOP
             +- NO ANSWER > RizzDial auto-dials again (native)
                            |
                            +- ANSWERED > outcome tag > STOP
                            +- NO ANSWER > tag ai-agent-no-answer > STOP for day

             NO VOICEMAIL LEFT.

DAYS 2-14 (business days only, 9am-5pm client timezone)
  Once per day, same pattern:
    Pre-call SMS > (60s) > RizzDial webhook (double_call:true)

STOP CONDITIONS
  ai-agent-booked, transferred, disqualified, do-not-contact, bad-number: STOP
  ai-agent-callback-requested: PAUSE until requested time
  Day 14 reached: tag ai-agent-exhausted, STOP
```

### Copy slots:

1. **Pre-Call SMS (Day 1)** — References what they signed up for. Agent identity. Under 160 chars. No link.
2. **Pre-Call SMS (Days 2+)** — Follow-up framing. "Blake again from Jeff's team, trying you one more time."
3. **Post-call disposition prompt** (in the Blake prompt, generated by /new-voice-ai-prompt) — Maps every scenario to one canonical tag.

### Rules:
- ALL calls fire only within business hours (9am-5pm client timezone, Mon-Fri)
- If opt-in hits outside business hours, queue for next business day 9am
- Do NOT generate separate "Call #1 / wait / Call #2" GHL steps — RizzDial's double_call handles it natively
- Pre-call SMS references what the lead opted in for, not generic "hey"
- NO voicemail scripts generated
- 14-day cap enforced with auto-stop tag

## AI CALL ON REPLY (ALWAYS GENERATED)

**GHL Name:** AI Call On Reply — P1 & P2
**Trigger:** Customer replies to ANY SMS or email from agent
**Purpose:** When a lead replies, agent calls them within 60 seconds via double_call.

### Architecture

```
LEAD REPLIES TO ANY AGENT SMS
  |
  v
P1 Check: Does contact have ai-agent-booked or ai-agent-do-not-contact?
  YES > STOP
  NO  > pass to P2
  |
  v
P2 Flow:
  1. Send pre-call SMS
  2. Wait 60 seconds
  3. Trigger RizzDial webhook (double_call:true)
  4. If answered > qualify > outcome tag
  5. If no answer on both dials > 2 retries at 15min intervals
```

### Copy slots:

1. **Pre-Call SMS (Reply Trigger)** — "Hey {{first_name}}, saw your message. Giving you a quick call right now." Under 75 chars.

### Rules:
- Check tags before triggering (skip booked or DNC)
- Highest-conversion trigger in the whole system — the reader just had the phone in their hand
- NO voicemail left on unanswered retries
